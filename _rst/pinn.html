PINN¶

Module for PINN

class PINN(problem, model, optimizer=<class 'torch.optim.adam.Adam'>, optimizer_kwargs=None, lr=0.001, lr_scheduler_type=<class 'torch.optim.lr_scheduler.ConstantLR'>, lr_scheduler_kwargs={'factor': 1, 'total_iters': 0}, regularizer=1e-05, batch_size=None, dtype=torch.float32, device='cpu', error_norm='mse')[source]¶

Bases: object

Parameters

  • problem (AbstractProblem) – the formualation of the problem.

  • model (torch.nn.Module) – the neural network model to use.

  • optimizer (torch.optim.Optimizer) – the neural network optimizer to use; default is torch.optim.Adam.

  • optimizer_kwargs (dict) – Optimizer constructor keyword args.

  • lr (float) – the learning rate; default is 0.001.

  • lr_scheduler_type (torch.optim.LRScheduler) – Learning rate scheduler.

  • lr_scheduler_kwargs (dict) – LR scheduler constructor keyword args.

  • regularizer (float) – the coefficient for L2 regularizer term.

  • dtype (type) – the data type to use for the model. Valid option are torch.float32 and torch.float64 (torch.float16 only on GPU); default is torch.float64.

  • device (str) – the device used for training; default ‘cpu’ option include ‘cuda’ if cuda is available.

  • int) error_norm ((str,) – the loss function used as minimizer, default mean square error ‘mse’. If string options include mean error ‘me’ and mean square error ‘mse’. If int, the p-norm is calculated where p is specifined by the int input.

  • batch_size (int) – batch size for the dataloader; default 5.
